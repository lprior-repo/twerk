# User Preferences

Update this when a preference is stable enough to shape future sessions.

## Workflow

- Use subagents for broad cleanup, verification, and risk triage.
- Push completed work when requested rather than leaving work stranded locally.
- Prefer high-certainty behavior claims backed by code reads and real verification where feasible.
- For release-readiness work, value manual end-to-end validation against the actual docs and shipped UX.

## Update Rules

- Keep only durable preferences that are likely to matter in later sessions.
- Replace stale bullets instead of appending conflicting ones.
- Do not store one-off instructions that were only task-local.
